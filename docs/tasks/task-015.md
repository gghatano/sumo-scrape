# Task-015: ログの充実

**Phase**: 5（品質）
**依存**: task-005, task-007, task-010
**ブロック**: なし

## 概要

運用時のデバッグ・監視に必要なログ出力を充実させる。

## やること

- 各フェーズでのログ出力を整備:
  - fetch: URL、HTTPステータス、リトライ回数、キャッシュヒット有無
  - parse: day、division、取組数、例外行数
  - write: 更新行数（fact/dim別）、対象event_id
- サマリーログ（実行完了時）:
  - 対象イベント一覧
  - fact合計行数、dim合計行数
  - 処理時間
- DEBUG レベルで詳細情報（個々の取組のパース結果等）

## 受入条件

- INFO レベルで実行の進捗・結果が把握できる
- DEBUG レベルで個別の取組レベルまで追跡できる
- エラー時にURL・ステータス・例外要点がログに残る
